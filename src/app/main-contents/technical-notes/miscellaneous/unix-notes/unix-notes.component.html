UNIX vs Linux

UNIX Linux
proprietary open-source

UNIX in itself is a complete OS as everything (all required applications tied together) comes from a single vendor.

Linux is a UNIX clone but it is just a kernel, not the complete OS.
This Linux kernel is generally packaged in Linux distributions which thereby makes it a complete OS.
  • macOS
  • Solaris
  • Debian
  • Ubuntu (based on Debian)
  • Alpine
  • Fedora
  • CentOS
Default shell is Bash Default shell is Bourne Shell (sh)
Not portable Portable and is booted from a USB Stick

apt vs apt-get vs dpkg

  • As compared to apt-get, apt is newer and has less features, but it is more user friendly and provides all necessary features.
  • In Linux, we have a package database, and normally we need to run apt update to update the package database before we install a package using apt install
  • For details about apt vs apt-get, refer to this tutorial.
  • APT DPKG
    APT (Advanced Package Tool) is a free software user interface for Debian and Debian-based Linux distributions that interacts with core libraries to handle program installation and removal. The package management system of the free operating system Debian and its many variations is based on the program DPKG.
    Packages from external repositories can be downloaded. Packages from external repositories cannot be downloaded.
    Has the ability to resolve dependencies Doesn’t have such ability.
    If you try to install a package whose dependencies are missing, APT downloads the dependencies. Dpkg will terminate and report missing dependencies if you try to install a package with it’s dependencies missing.
  • In short, apt is "higher-level" package management tool whereas dkpg is "lower-level".
