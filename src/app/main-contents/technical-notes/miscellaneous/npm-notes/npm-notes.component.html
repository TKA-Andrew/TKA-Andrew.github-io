package.json vs package-lock.json

The following are the notes summarized from this Medium post.

  • package.json is the versioning file used to install different packages in a Node.js project
  • Every time you install any package using npm install <package-name> --save, it will install the latest version of that package in your project.
  • Then, the package.json of your project will be updated by adding the dependency of the newly installed package with a carat(^) sign.
  • The carat (^) sign indicates that it will support any higher version within the same major version.
  • For example, if you installed a lodash-es at version 4.17.21, then your package.json will have "lodash-es": "^4.17.21". If few months later the latest lodash-es package is 4.20.2, and when someone clones your package and run npm install, the lodash-es version installed will be 4.20.2
  • Hence, to make sure the project is cloned and installed with exactly same dependencies, we need to have package-lock.json to lock the versions of packages we used.
