Communication Protocols Notes

Introduction

  • Communication protocols are the languages used by electronic devices to communicate.
  • Both sides should speak the same language in order to communicate.
  • Communication protocols can be categorized into:
    • Serial Communication
      • The bits are sent one by one through a single wire.
    • Parallel Communication
      • The bits of data are sent all at the same time, each through a single wire.

Serial Communication

SPI (serial peripheral interface)

  • Detailed reference
    • Source of image: https://www.researchgate.net/figure/Exact-Cell-Decomposition-Latombe-1991_fig3_235929101
    • MOSI (Master Output/Slave Input) - Line for the master to send data to the slave.
    • MISO (Master Input/Slave Output) - Line for the slave to send data to the master.
    • SCLK (Clock) - Line for the clock signal.
    • SS/CS (Slave Select/Chip Select) - Line for the master to select which slave to send data to.
  • Miscellaneous notes
    • Data sent from the master to the slave is usually sent with the most significant bit first.
    • The data sent from the slave back to the master is usually sent with the least significant bit first.
  • Summary
    • Wires used 4
    Maximum speed Up to 10Mbps
    Synchronous or Asynchronous Synchronous
    Advantages
  • No start and stop bits, so the data can be streamed continuously without interruption
  • No complicated slave addressing system like I2C
  • Separate MISO and MOSI lines, so data can be sent and received at the same time
  • Higher data transfer rate than I2C
    		•
    Disadvantages
  • Uses four wires (I2C and UARTs use two)
  • No acknowledgement that the data has been successfully received
  • No form of error checking like the parity bit in UART
  • Only allows for a single master
    		•

UART (Universal Asynchronous Receiver/Transmitter)

  • Detailed reference
    • Source of image: https://www.circuitbasics.com/basics-uart-communication/
  • Miscellaneous notes
    • UART is NOT a communication protocol like SPI and I2C, but a physical circuit in a microcontroller, or a stand-alone IC.
    • The transmitting UART converts parallel data into serial form and then transmits it in serial to the receiving UART.
    • UART transmitted data is organized into packets.
      • Source of image: https://www.circuitbasics.com/basics-uart-communication/
    • Baud rate is a measure of the speed of data transfer, expressed in bits per second (bps)
    • The baud rate between the transmitting and receiving UARTs can only differ by about 10% before the timing of bits gets too far off.
  • Summary
    • Wires used 2
    Maximum speed Any speed up to 115200 baud, usually 9600 baud
    Synchronous or Asynchronous Asynchronous
    Advantages
  • Only uses two wires
  • No clock signal is necessary
  • Has a parity bit to allow for error checking
  • The structure of the data packet can be changed as long as both sides are set up for it
    		•
    Disadvantages
  • The size of the data frame is limited to a maximum of 9 bits
  • Only allow single master and single slave
  • The baud rates of each UART must be within 10% of each other
    		•

I2C (Inter-Integrated Circuit)

  • Detailed reference
    • Source of image: https://www.circuitbasics.com/basics-of-the-i2c-communication-protocol
    • SDA (Serial Data) - The line for the master and slave to send and receive data.
    • SCL (Serial Clock) - The line that carries the clock signal.
  • Miscellaneous notes
    • Data is transferred in messages.
      • Source of image: https://www.circuitbasics.com/basics-of-the-i2c-communication-protocol
  • Summary
    • Wires used 2
    Maximum speed 100kbps - 5Mbps
    Synchronous or Asynchronous Synchronous
    Advantages
  • Only uses two wires
  • Supports multiple masters and multiple slaves
  • ACK/NACK bit gives confirmation that each frame is transferred successfully
  • Hardware is less complicated than with UARTs
    		•
    Disadvantages
  • Slower data transfer rate than SPI
  • The size of the data frame is limited to 8 bits
