Dynamic Programming

Introduction

  • Dynamic Programming is the method of solving a problem by breaking it down into smaller easier problems.
  • For us to be able to apply Dynamic Programming, it has 2 properties:
    • Optimal substructure
      • Finding the solution to each its subproblems and joining the individual solutions can lead to optimal solution of the original problem
    • Overlapping subproblems
      • The solutions to the subproblems are mutually dependent
      • Also known as 'state transition' in this note
  • What we need in DP solution:
    • base case
    • state transition
    • choices for state transition
    • meaning of "dp" variable (normally is 1D or 2D vector, or even recursive function)
  • There is no standard sequence of which of the above items should be determined first, it all depends on the definition of your "dp" variable.
  • State compression can be done to compress the dp vector (from 2D to 1D, or from 1D to a few variables) if the state transition only relies on neighbor elements
  • Normally problems that relate to finding maximum, minimum, greatest, longest etc are possibly related to DP (as normally the problem fulfills the 2 properties of Dynamic Programming)
  • Questions related to subsequences also possibly related to DP
  • The content here will try to focus on bottom-up approach of DP only, will try to use the same pattern of solution
