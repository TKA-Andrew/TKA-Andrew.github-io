HTML Basics

  • The <!DOCTYPE html> at first line of html file tells browser that it is a HTML file.
  • Take note that certain types of tags are not supported by certain type or version of browsers, to check, use caniuse.com
  • The html elemetn includes a head element and a body element:
    • The head element contains information about the page such as its title.
    • The body element includes the structure and content of the page.
  • Head
    • meta element - charset
      • Types of charset
        • ASCII
          • ASCII can only represents characters in English Language
        • UTF-8
          • This can represent almost all characters in the world.
    • meta element - viewport
      • The viewport is the visible area of a webpage.
      • Viewport element allows us to define the inital width and scale.
    • Example:
      •                                 
    <html>
        <meta charset="UTF-8"/>
        <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
    </html>
  • Text
    • Use <em</em> for emphaized text
    • Use <strong</strong> for important/serious/urgent content
    • <b> and <i> can make text bold and italic respectively, but they should not be used as it should be the responsibilty of CSS to do the styling.
  • Entities
    • Sometimes you might want to display some symbols, you may refer to the HTML Entities Reference Chart
  • Hyperlink
    • href can be used for relative or absolute reference to html files
    • href can also be used to "scroll" to another fragment.
    • Adding "download" property to a hyperlink of an image can also make it downloadable.
    • Example: 
                                      
    <body>
        <a href="company/about.html">About</a>
        <a href="images/profile.jpg" download>Profile</a>
        <a href="#section-css">CSS</a>
        <a href="https://google.com">Google</a>
        <h2>HTML</h2>
        <h2 id="section-css">CSS</h2>
        <a href="#">Jump to top</a>
    </body>
  • Image, Video and Audio
    • You may get free images from unslash.com
    • You may get free videos from pexels.com
    • To include video in your html, use <video> tag, and you can use these properties also:
      • controls
      • autoplay
      • loop
    • For audio, use <audio>, it has the same properties as video tag as well.
  • List
    • Types of list:
      • unordered list - ul + li
      • ordered list - ol + li
      • description list - dl + dt + dd
  • Table
    • Example:
      •                                 
    <table>
        <thead>
            <tr>
                <th colspan="2">Expenses</th>
            </tr>
            <tr>
                <th>Category</th>
                <th>Amount</th>
            </tr>
        </thead>
        <tbody>
            <tr>
                <td>Marketing</td>
                <td>$ 500</td>
            </tr>
            <tr>
                <td>Accounting</td>
                <td>$ 200</td>
            </tr>
        </tbody>
        <tfoot>
            <tr>
                <th>Total</th>
                <th>$ 700</th>
            </tr>
        </tfoot>
    </table>
  • Container
    • The most common containers are <div> and <span> elements.
    • div is block-element whereas span is inline-element.
    • Block-level element always starts on a new line, and fill up the entire available space. You may set the width through CSS to reduce its width.
    • Inline element only takes up the width it needs.
    • div and span are general purpose containers. Other than general purpose containers, there are also semantic elements which make the elements more descriptive.
      • <article>
      • <figure>
      • <mark>
      • <time>

CSS Basics

  • Basics
    • Basic selectors:
      • type
        • For example: p { color: black }
      • id
        • For example: p#special_remark { color: purple }
      • class
        • For example: p.product_description { color: grey }
      • property
        • For example: a[href^="https"][href$=".com"] { color: blue }
    • There are also relation selectors, they could make the markup cleaner but can be fragile as well.
    • Pseudo-class Selectors & Pseudo-element Selectors are very helpful too.
      • Example of Pseudo-class Selectors
        • p:hover
        • a:visited
        • tr:nth-child(odd)
      • Example of Pseudo-element Selectors
        • ::selection
        • p::first-line
        • p::first-letter
    • The level of selector specificity (Weight) is arranged as follows, from highest to lowest:
      1. ID Selector
      2. Class & property Selectors
      3. Element Selectors
    • The level of specificity can be higher by combining the types of selectors. For example: "p .info #productID" has much higher specificity than "#productID"
    • Inheritance
      • Some CSS properties by default, inherit values from parents.
      • To avoid inherit or force the selector the inherit, use "initial" and "inherit". For example: 
        •                                         
    strong {
        color: initial;
        border: inherit;
    }
    • To avoid the selectors being overrided, we can use the !important keyword, but take note that this is a bad practice.
    • Take note that when defining properties of a selector, the bottom one will always replace the upper one if there is conflict. For example, the bottom-top property below will be replaced by the border property: 
      •                                 
    .box {
        border-top: 10px dashed blue;
        border: 5px solid red;
    }
    • Colors
      • rgb
      • rgba
      • 6 digit HEX
      • 3 digit HEX
      • hsl
      • hsla
    • Gradients
    • Borders
    • Shapes
    • To embed css style:
      1. Move all styles definition into a .css file
      2. Add this line into head of html: <link rel="stylesheet" href="styles.css"/>
    • Normalizing css
      • HTML is rendered differently in different browsers. To address this problem, there is a css file called normalize.css
      • Simply link the normalize.css file into your html file to use it.
  • img
    • object-fit
      • To fit image into a fixed width and height.
      • Example of options are: scale-down, cover, fill
  • list
    • list-style-types
      • square
  • table
    • border: line-width line-type line-color
    • border-collapse: collapse

Layouts

  • Box Model
    • Content, Padding, Border, Margin:
      • Source of image: https://www.quora.com/What-are-the-differences-between-margin-and-padding
  • Sizing Elements
    • Imagine if you define the following html and css:
      •                                 
    <div class="box"></div>

    .box {
        height: 100px;
        width: 100px;
        background-color: purple;
        padding: 20px 10px 20px 10px; /*top right bottom left*/
        border: 10 px solid black;
    }
      You will get a div box with size greater than 100x100.
    • This is because the default box-sizing property is content-box, so the height and weight only consider the content.
    • To make the entire box including padding and border to be within 100px x 100px, simply set the box-sizing property to border-box.
    • Take note that if you use inline element (such as span), inline elements dont respect width and height. You need to change the display property from 'inline' to 'inline-block'
  • Overflow
    • Overflow happens when your content is bigger than your container.
    • The default value of overflow property is visible.
    • Some other options of values:
      • hidden
        • hide all overflow content
      • scroll
        • display a scroll bar no matter there is overflow or not
      • auto
        • only display scrollbar when there is overflow
    • You can control the overflow in 2 directions using overflow-x and overflow-y.
  • Measurement units
    • Absolute measurement units
      • px
      • pt
      • in
      • cm
      • mm
    • Relative measurement units
      • %
        • relative to the size of the container
      • vw
        • relative to the viewport
      • vh
        • relative to the viewport
      • em
        • Relative to parent element's font size
        • 10em means 10 times of font size
      • rem
        • Relative to root(HTML)font size
        • So if you wish to scale the element's size based on the root size, no matter what the parent size is, use REM.
    • By default, the body element is block element, and hence it always take up 100% width of the browser. BUT take note that, the body element does not have a height property, and hence by default the height of body is zero. HENCE, if you want to do something relative to view port (the visible area of a webpage), you should use vw and vh.
  • Positioning
    • By default, the value of position property is 'static', means they are not positioned.
    • Other options of position property:
      • relative
        • Relative to its normal position (its static position)
        • Then, you can start to set left, right, top, bottom properties
        • The positions of other elements will not be affected by this element.
      • absolute
        • Relative to the parent.
        • If one of your elements want to be absolute, your parent container should set the position property to be relatives.
        • If a container contains an element with absolute positioning, from the container's point of view, the absolute element does not exist. And hence, the absolute positioned element's positioning is relative to parent container.
      • fixed
        • Relative to view port
        • Hence, to have a navigation bar that is always stay on top of view port:
          •                                                 
    .navigation-bar {
        position: fixed;
        top: 0;
    }
        • If you set width and height to auto, the browser will decide its width and height.
    • By default, z-index value is 0. Setting it to higher value means it will be "closer" to us, it can be negative value also.
  • Floating elements
    • float property
      • By setting an element to float, for example 'float: left;', it will make that element float to left side of the container. And then all the subsequent elements will float around it.
      • However, take note that parent elements dont see floating elements.
    • clear property
      • clear property can be used to clear floats.
      • A typical use ase is as below:
        •                                         
    <article class="tweet clearfix">
        <div class="avatar"></div>
        <p>Lorem ipsum dolor sit amet.</p>
    </article>

    .clearfix::after {
        content: '';
        display: block;
        clear: both;
    }
  • Flexbox (Flexible Box Layout)
    • Used for laying out elements in one direction (row or column).
    • To use flexbox, you need to set display: flex; for the parent element.
    • Even though we set display: flex;, the container is still a block-level element.
    • You can use flex-direction to control the direction, by default it is row.
    • To use flexbox, you must know the flexbox axes:
      • Source of image: https://www.samanthaming.com/flexbox30/4-flexbox-axes/
    • justify-content property
      • To align item along the main axis
      • Default value: flex-start
    • align-items property
      • To align item along the cross axis
      • Default value: flex-start
    • flex-wrap property
      • Take note that, let's say you have a flex container and it contains multiple boxes, even if you set the width property of the boxes, their width will be modified to ensure all the elements fit in one line.
      • The default value of flex-wrap is nowrap
      • If you set flex-wrap: wrap;, then the items in the flex container can keep their own width and height.
    • align-content property
      • This property only works if you have multiple lines in a flex container.
      • When you have multiple lines in a flex container, for example due to flex-wrap, you can use align-content to align all the lines as a whole.
    • align-self and justify-self property
      • If you want override the align-items or justify-content property set by parent container, you can use align-self or justify-self.
    • Sizing items:
      • The following properties can be used to size items (cannot be used for container)
        • flex-basis property
          • Default value is auto, which means browser sets the width and height based on the width and height properties.
          • This will override width or height based on the flex main-axis:
            • If flex-direction is set to row, it overrides the width
            • If flex-direction is set to column, it overrides the height
        • flex-grow property
          • This determines the grow factor.
        • flex-shrink property
          • This determines the shrink factor.
        • flex property
          • It is a combination of flex-grow, flex-shrink and flex-basis.
  • Grid
    • Flexbox allows us to lay out elements in a row or a column.
    • Grid allows us to lay out elements in both rows and columns.
    • To use grid, you need to set display: grid; for the parent element.
    • grid-template-rows
      • To define the height of each grid row.
      • For example: grid-template-rows: 100px 100px 100px; will define that there will be 3 rows in the grid, and each of them is 100px height.
    • grid-template-columns
    • justify-items property
      • To align items in each grid cell along the horizontal axis
      • Default value is stretch, but it will not be applied if the item has already defined height and width.
    • align-items property
      • To align items in each grid cell along the vertical axis
      • Default value is stretch, but it will not be applied if the item has already defined height and width.
    • justify-content property
      • To align the entire grid along the horizontal axis
    • align-content property
      • To align the entire grid along the vertical axis
    • fr measurement unit
      • Grid layout has a special relative measurement unit called 'fr', it is called fraction.
      • Unlike % which is relative to the size of container, fr is relative to the size of the remaining space.
      • For example, if width of container is 200px, and the grid-template-columns: 100px 20fr 80fr means the width of second column will be 20px and width of third column will be 80px.
      • Other than fr, we can also use auto to let the browser automatically fill up the remaining space.
    • row-gap and column-gap properties
      • To adjust the gaps between grid cells
    • grid-column and grid-row properties
      • To implicitly place the item at specific row and column.
      • By default, each item takes only one cell.
      • If you want it to take more than one cell, use span. For example: grid-column: 1 / span 2
    • grid-template-areas and grid-area properties
      • With grid-template-areas, we can give names to each grid cell. For example: 
        •                                         
    grid-template-areas: 
        "header     header"
        "sidebar    main"
        "footer     footer";
      • With the names defined, you can now use grid-area property to place it in named areas. For example: grid-area: footer
  • Hiding elements
    • Using display property:
      • If you set display: none;, no space will be allocated for the item and it will not be displayed as well.
    • Using visibility property:
      • If you set visibility: hidden;, the space will still be allocated for the item, but it is just not visible.
  • Media Queries
    • When designing responsive web, normally people prefer to build the mobile version first, then only do adjustment for desktop version.
    • With media queries, we can set different class style based on width and height.
    • For example:
      •                                 
    .responsive-container {
        display: flex;
        flex-direction: column;
        padding: 10px;
    }

    @media screen and (min-width: 800px) {
        .responsive-container {
            flex-direction: row;
        }
    }

Typography

  • 3 main types of fonts:
    • Serif
      • More professional and serious
      • Examples:
        • Times New Roman
    • Sans-serif
      • More playful and modern
      • Examples:
        • Arial
        • Roboto
    • Monospace
      • Ideal for displaying code
      • Examples:
        • Courier
        • Ubuntu
        • Consolas
  • Web Safe Fonts
    • Web Safe Fonts are fonts that are available on most computers out there.
  • Font Stack
    • A font stack consists of multiple fonts, and it will try to use the first font in the stack if it is available, else it will look for the next one. The last one in the stack should be a generic font type, such as serif, sans-serif or monospace.
    • Example of setting font-family using font stack:
      •                                 
    font-family: Georgia, 'Times New Roman', serif;
  • font-weight
    • Range is between 100 - 900
    • normal = 400
    • bold = 700
  • font-size
    • The common best practice of setting font size is by setting it relative to root font size by using 'rem'. This applies to margin etc as well.
    • This is so that we can easily scale the font size according to different screen size. For example: 
      •                                 
    body {
        font-size: 1rem;
    }
    h1 {
        font-size: 2rem;
    }
    @media screen and (min-width: 400px) {
        html {
            font-size: 130%;
        }
    }
  • line-height
    • The general rule of thumb is to set the line-height to 1.5 times of font-size.
    • To make it always relative to font size, no matter how we change the font size, we can just set it to 1.5, without px without rem etc. 
      •                                 
    body {
        font-size: 1rem;
        line-height: 1.5;
    }
  • text-align
    • options: left, right, center, justify
  • text-indent
    • This is to add indentation to the first line of our text.
  • text-decoration
    • options: underline, line-through
  • text-transform
    • options: capitalize, lowercase, uppercase
  • splitting text to columns
    • Example:
      •                                 
    p {
        column-count: 2;
        column-gap: 1rem;
        column-rule: 3px dotted #999;
    }
  • setting nowrap to long text and shows "..." at the end
    • Example:
      •                                 
    p {
        white-space: nowrap;
        border: solid;
        overflow: hidden;
        text-overflow: ellipsis;
    }
