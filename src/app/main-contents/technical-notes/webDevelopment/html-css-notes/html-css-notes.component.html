HTML Notes

Basics

  • The <!DOCTYPE html> at first line of html file tells browser that it is a HTML file.
  • Take note that certain types of tags are not supported by certain type or version of browsers, to check, use caniuse.com
  • The html elemetn includes a head element and a body element:
    • The head element contains information about the page such as its title.
    • The body element includes the structure and content of the page.
  • Head
    • meta element - charset
      • Types of charset
        • ASCII
          • ASCII can only represents characters in English Language
        • UTF-8
          • This can represent almost all characters in the world.
    • meta element - viewport
      • The viewport is the visible area of a webpage.
      • Viewport element allows us to define the inital width and scale.
    • Example:
      •                                 
    <html>
        <meta charset="UTF-8"/>
        <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
    </html>
  • Text
    • Use <em</em> for emphaized text
    • Use <strong</strong> for important/serious/urgent content
    • <b> and <i> can make text bold and italic respectively, but they should not be used as it should be the responsibilty of CSS to do the styling.
  • Entities
    • Sometimes you might want to display some symbols, you may refer to the HTML Entities Reference Chart
  • Hyperlink
    • href can be used for relative or absolute reference to html files
    • href can also be used to "scroll" to another fragment.
    • Adding "download" attribute to a hyperlink of an image can also make it downloadable.
    • Example: 
                                      
    <body>
        <a href="company/about.html">About</a>
        <a href="images/profile.jpg" download>Profile</a>
        <a href="#section-css">CSS</a>
        <a href="https://google.com">Google</a>
        <h2>HTML</h2>
        <h2 id="section-css">CSS</h2>
        <a href="#">Jump to top</a>
    </body>
  • Image, Video and Audio
    • You may get free images from unslash.com
    • You may get free videos from pexels.com
    • To include video in your html, use <video> tag, and you can use these attributes also:
      • controls
      • autoplay
      • loop
    • For audio, use <audio>, it has the same attributes as video tag as well.
  • List
    • Types of list:
      • unordered list - ul + li
      • ordered list - ol + li
      • description list - dl + dt + dd
  • Table
    • Example:
      •                                 
    <table>
        <thead>
            <tr>
                <th colspan="2">Expenses</th>
            </tr>
            <tr>
                <th>Category</th>
                <th>Amount</th>
            </tr>
        </thead>
        <tbody>
            <tr>
                <td>Marketing</td>
                <td>$ 500</td>
            </tr>
            <tr>
                <td>Accounting</td>
                <td>$ 200</td>
            </tr>
        </tbody>
        <tfoot>
            <tr>
                <th>Total</th>
                <th>$ 700</th>
            </tr>
        </tfoot>
    </table>
  • Container
    • The most common containers are <div> and <span> elements.
    • div is block-element whereas span is inline-element.
    • Block-level element always starts on a new line, and fill up the entire available space. You may set the width through CSS to reduce its width.
    • Inline element only takes up the width it needs.
    • div and span are general purpose containers. Other than general purpose containers, there are also semantic elements which make the elements more descriptive.
      • <article>
      • <figure>
      • <mark>
      • <time>

CSS Notes

  • Basics
    • Basic selectors:
      • type
        • For example: p { color: black }
      • id
        • For example: p#special_remark { color: purple }
      • class
        • For example: p.product_description { color: grey }
      • attribute
        • For example: a[href^="https"][href$=".com"] { color: blue }
    • There are also relation selectors, they could make the markup cleaner but can be fragile as well.
    • Pseudo-class Selectors & Pseudo-element Selectors are very helpful too.
      • Example of Pseudo-class Selectors
        • p:hover
        • a:visited
        • tr:nth-child(odd)
      • Example of Pseudo-element Selectors
        • ::selection
        • p::first-line
        • p::first-letter
    • The level of selector specificity (Weight) is arranged as follows, from highest to lowest:
      1. ID Selector
      2. Class & Attribute Selectors
      3. Element Selectors
    • The level of specificity can be higher by combining the types of selectors. For example: "p .info #productID" has much higher specificity than "#productID"
    • Inheritance
      • Some CSS properties by default, inherit values from parents.
      • To avoid inherit or force the selector the inherit, use "initial" and "inherit". For example: 
        •                                         
    strong {
        color: initial;
        border: inherit;
    }
    • To avoid the selectors being overrided, we can use the !important keyword, but take note that this is a bad practice.
    • Take note that when defining properties of a selector, the bottom one will always replace the upper one if there is conflict. For example, the bottom-top property below will be replaced by the border property: 
      •                                 
    .box {
        border-top: 10px dashed blue;
        border: 5px solid red;
    }
    • Colors
      • rgb
      • rgba
      • 6 digit HEX
      • 3 digit HEX
      • hsl
      • hsla
    • Gradients
    • Borders
    • Shapes
    • To embed css style:
      1. Move all styles definition into a .css file
      2. Add this line into head of html: <link rel="stylesheet" href="styles.css"/>
    • Normalizing css
      • HTML is rendered differently in different browsers. To address this problem, there is a css file called normalize.css
      • Simply link the normalize.css file into your html file to use it.

      • Example of options are: scale-down, cover, fill

      • square
  • table
