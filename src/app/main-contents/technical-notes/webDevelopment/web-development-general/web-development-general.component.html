How the Web works?

  • The front-end is the part that is loaded inside a web browser on a client machine. It is the part the user sees and interacts with.
  • The back-end is the part that runs on a web server, and is responsible for data processing, validating, business rules, etc.
  • URL (Uniform Resource Locator) is a way to locate a resource on our internet. A resource can be a webpage, an image, a video, a PDF, etc.
  • When someone types lets say www.abc.com in the browser and press Enter, the browser sends a HTTP request to the web server. Then, the webserver takes the request and returns a response to the client. There are 2 ways of returning responses:
    • Generates the page on the server, and then return an html document to the client
    • Return the data needed on the requested page, and let the client generate the page
  • Second option above is more preferable as it frees up the server, so the server can serve more clients.
  • With the second option being implemented, the server becomes an gateway to the data. On the server, we can provide different endpoints (API) to let client get different data. For example, /products to get list of products.
  • Back to the HTTP request,
    • The communication between client and server is based on HTTP protocol.
    • Here is a simplified example of an HTTP message, and basically it means "GET a file called index.html using HTTP version 1.1":
      •                                 
    GET /index.html HTTP/1.1
    Host: www.abcdefg.com
    Accept-Language: en-us
    • index.html normally represents the home page of websites
    • Here is a simplified example of HTTP response:
      •                                 
    HTTP/1.1 200 OK
    Date: 13 Oct 2022 00:15
    Content-Type: text/html

    <!DOCTYPE html>
    <html>
        ...
    </html>
    • As the browser reads the HTML document, it constructs a DOM (Document Object Model) which represents an object/element in HTML documents. The elements include paragraph of text, images, links etc.
    • If the HTML document contains references to other resources such as images, and each of this resource has its URL, the browser will further send HTTP requests to fetch the resources.

What is jQuery?

  • Take note that: The recent trend is towards migrating jQuery to Vanilla Javascript. Vanilla Javascript here means "Core Javascript" or "Plain Javascript" without unnecessary libraries. The trend of this migration is because:
    • Whatever jQuery can do, it can be done using Javascript
    • jQuery has lots of dependencies, and the browser has to load a lot of files just to do something that Javascript can do.
  • jQuery is a lightweight, "write less, do more" Javascript library.
  • jQuery is the most-know skill after having a basic knowledge about HTML and CSS.
  • jQuery is so useful that it has the folloing features:
    • HTML/DOM manipulation
      •                                 
    // FOR EXAMPLE
    $("p").append("Some appended text.");
    $(".productDescription").prepend("Some prepended text.");
    • CSS manipulation
      •                             
    // FOR EXAMPLE
    $("#pageTitle").css({"color": "black", "font-style": "italic", "font-weight": "bold"});
    • HTML event methods
      •                                 
    // FOR EXAMPLE
    $("#button1").click(function() {
        $("#div1").toggle();
    })
    • Effects and animation
      •                                 
    // FOR EXAMPLE
    $("#div1").slideUp();
    $("#div2").animate({height: '300px', opacity: '0.5'}, "slow");
    • AJAX
      •                                 
    // FOR EXAMPLE
    $("#div1").load("demo_test.txt");
    • Utilities

ES Modules

ES Modules is the ECMAScript (the scripting language that forms the basis of JavaScript) standard for working with modules.

Node.js has been using the CommonJS standard for years but there was no concept of modules in Javascript, and <script> tags were used to load multiple Javascript files. Therefore, the module system support is finally standardized in ES6 and then browsers started to implement it. Now Chrome, Safari, Edge and Firefox (starting from version 60) all support ES modules.

Some important notes:

  • ES6 modules are automatically strict-mode code, even if you don’t write "use strict"; in them
  • Everything inside an ES6 module is private by default
  • Public variables, functions and classes are exposed using export
  • Exposed modules are called into other modules using import
  • The type="module" directive must be included in the script tag in order to include modules
  • You need to add nomodule directive in script tag for browsers without module support
  • Regular script tags can fetch scripts on other domains but modules are fetched using cross-origin resource sharing (CORS). Modules on different domains must therefore set an appropriate HTTP header, such as Access-Control-Allow-Origin: *
  • Modules are deferred, and only run after a document is loaded

CommonJS

The CommonJS module specification is the standard used in Node.js for working with modules.

The npm ecosystem is built upon this CommonJS format.

In CommonJS, modules are loaded synchronously, and processed in the order the JavaScript runtime finds them.

CommonJS was born with server-side Javascript in mind whereas ES Modules were introduced for client-side Javascript.

ES Modules vs CommonJS

Take note that Node 14 introduces ECMAScript Modules, hence, you can use both ES Modules and CommonJS modules in Node 14. For this, you need to know how to make the files being treated as ECMAScript modules or CommonJS modules by referring to this article.

Import examples:

                
    // IN ES module standard
    import {map, reduce} from 'lodash'
    import _ from 'lodash' // Default Import
    import {cloneDeep as clone} from 'lodash' // Named Imports
    
    // IN CommonJS standard
    const {map, reduce} = require('lodash')
    const _ = require('lodash') // Default Import
    const {cloneDeep: clone} = require('lodash'); // Named Imports

Export examples:

                
    // IN ES module standard
    export default 1
    export default 'foo'
    export default { foo: 'bar' }
    export default ['foo', 'bar']
    export default function foo () {}
    export const fooString = 'foo' // Name Exports
    export const consoleABC = function () { console.log('ABC') } // Named Exports
    
    // IN CommonJS standard
    module.exports = 1
    module.exports = 'foo'
    module.exports = { foo: 'bar' }
    module.exports = ['foo', 'bar']
    module.exports = function foo () {}
    module.exports.fooString = 'foo' // Name Exports
    module.exports.consoleABC = function () { console.log('ABC') } // Named Exports
