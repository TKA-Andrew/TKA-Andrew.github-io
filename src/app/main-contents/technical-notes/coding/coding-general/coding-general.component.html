SOLID Principles

Introduction

SOLID Principles are the acronym for:

  • S - Single-responsiblity Principle
  • O - Open-closed Principle
  • L - Liskov Substitution Principle
  • I - Interface Segregation Principle
  • D - Dependency Inversion Principle

These are the principles for developing maintainable and extensible software project.

Asynchronity in Programming Languages

Introduction

Asynchronous means that things can happen independently of the main program flow.

C, Java, C#, PHP, Go, Ruby, Swift, and Python are all synchronous by default. Some of them handle async by using threads, spawning a new process.

Javascript is also synchronous and single-threaded as well but people always confused that Javascript is asynchronous in nature. Javascript can behave as if it is asynchronous when there are some ajax calls, events or timers being used.
